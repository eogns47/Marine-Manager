## PR 규칙
* main : 배포용 branch
* test : 배포 전 테스트 서버용 branch
* feature: 개발용 branch
* fix : 오류 수정용 branch
---
* PR은 fix또는 feature로 부터 main 으로 날리는 방식으로 진행
* main으로 바로 push 하는 일은 ❌
* 커밋 메세지는 'fix : ~~~ '형식으로 작성하고, pull 시 변경되는 사항들을 모두 자세히 작성
